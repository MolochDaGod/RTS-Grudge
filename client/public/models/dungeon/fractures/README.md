# Pre-fractured Decor GLBs

Pre-sliced GLB variants of destructible dungeon decor. The shatter
system spawns the child meshes here as physics fragments instead of
falling back to generic colored cuboids. Each chunk re-uses the source
prop's own materials (PBR atlas textures for fantasy props,
vertex-colored palettes for KayKit), so smashed fragments match the
intact prop's look.

## Authoring workflow

The GLBs are generated by
[`scripts/build-fracture-glbs.cjs`](../../../../../scripts/build-fracture-glbs.cjs).
The script loads each source prop, slices its triangles into chunks
based on each triangle's centroid (fully deterministic — no RNG), and
re-exports a small GLB whose meshes share the source prop's materials.
Image textures are kept as relative URIs back to the source pack folder
so the heavy PBR atlases are loaded once and shared with the intact
prop.

To add or revise a fracture asset:

1. Edit the per-prop entry in `scripts/build-fracture-glbs.cjs`.
2. Run `node scripts/build-fracture-glbs.cjs` from the project root.
3. (If new) register the file in `FRACTURE_ASSETS` inside
   `client/src/game/dungeon/DungeonFractureAssets.tsx`.

If you'd rather drop a hand-modeled GLB instead, just place the file
here and add the registry entry. Loader expectations:

- One mesh per chunk (top-level or nested — the loader traverses).
- Each mesh in its final position relative to the intact prop. The
  loader auto-scales the whole scene to the in-game prop height and
  re-centers each chunk's geometry around its own origin.
- Aim for ≤ 10 chunks per file. Anything over the cap is dropped
  largest-first.

Decor types not listed in `FRACTURE_ASSETS` (or whose GLB fails to
load) automatically fall back to the existing colored-cuboid fragments.
